Introduction

Vishwakarma Build System is built completely on top of Node JS. It is designed to be super fast and hassle-free. The project is fully Open Source and can be found here

The name Vishwakarma was chosen because Vishwakarma is the celestial architect and builder of the devas as per Hindu Mythology

These docs are written to get you up and running with the least effort. Click on any topic on the left to skip ahead, or click on Next below

Getting Started

If you are running Vishwakarma for the first time, you have to create an administrator account. To create the first administrator account, click here to open the bootstrap page

Provide a username and a password and create the administrator account. This account is very important. Please do not lose the username and password

Configuration

Vishwakarma comes with a config.default file which can be used to set configurations. You can alternately use your own config.local file to override default configurations. Configuration files allow you to set various server URLs and ports, etc.

Here is a sample configuration file. It shows that the server is running on port 8080, MongoDB is running on localhost on port 27017. If you are running Vishwakarma behind another server, say Nginx (via proxy_pass), you can change the local server to whatever is appropriate. Make sure your Nginx socket configuration is appropriate 

{
    "server": {
        "port": 8080
    },
    "mongo": {
        "server": "localhost",
        "port": 27017
    },
    "local": {
        "server": "http://localhost:8080"
    },
    "logs": {
        "save": true,
        "path": "/tmp/",
        "retain": 10
    },
    "account": {
        "create": "true"
    }
}

Non-Administrator Accounts

It is possible to create Non-Administrator accounts from the Home page. However, these accounts will not have ANY privileges in the beginning. They also do not belong to any groups. The administrator can assign users to groups and give privileges to groups

Administrator

The admin has the following powers

Manage User & Groups

The admin can access the "Manage User & Groups" section

The admin can create groups

The admin can assign users to groups or remove them from groups

Create & Manage Projects

In Vishwakarma, only admins can create projects

Admins can provide privileges to other groups (see project privileges)

Admins get all privileges to all projects by default

Creating Projects

In Vishwakarma, only admins can create projects

To create a new project, click on 'New Project' on the home screen

Provide a project name and the code of the project. The code is always a shell script

Save the project

Chaining Projects

On successful execution of a project, you can automatically execute another project. This is called as Chaining Projects

In Vishwakarma, it is as simple as selecting a project from the drop down

Scheduling Projects

You can schedule projects using standard cron notation

The Kaala-Bhairava Visual Cron Tool is currently under development. Kaala-Bhairava is the god of time

Manage Groups & Permissions

This section lets you manage permissions to different groups. Vishwakarma has the following permissions for projects

  • Hide - Projects can be hidden from groups. Hidden projects are completely non-existent to the groups in all ways
  • Read - Groups with Read privilege can see project's existence and the status of the project (running, completed, aborted)
  • Edit - Groups with Edit privilege can read and edit code of projects
  • Run - Groups with Run privilege can run projects
  • Abort - Groups with Abort privilege can abort running projects
  • See Logs - Groups with See Logs privilege can see logs of projects, running and historic

Logs

Vishwakarma logs all executed projects
